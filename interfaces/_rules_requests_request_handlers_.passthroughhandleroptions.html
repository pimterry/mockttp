Optional beforeRequest

beforeRequest: undefined | ((req: CompletedRequest) => MaybePromise<CallbackRequestResult>)
  • Defined in src/rules/requests/request-handlers.ts:477

A callback that will be passed the full request before it is passed through, and which returns an object that defines how the the request content should be changed before it's passed to the upstream server.

The callback should return an object that definies how the request should be changed. All fields on the object are optional. The possible fields are:

  • method (a replacement HTTP verb, capitalized)
  • url (a full URL to send the request to)
  • response (a response callback result: if provided this will be used directly, the request will not be passed through at all, and any beforeResponse callback will never fire)
  • headers (object with string keys & values, replaces all headers if set)
  • body (string or buffer, replaces the body if set)
  • json (object, to be sent as a JSON-encoded body, taking precedence over body if both are set)

Optional beforeResponse

beforeResponse: undefined | ((res: PassThroughResponse) => MaybePromise<CallbackResponseResult>)
  • Defined in src/rules/requests/request-handlers.ts:494

A callback that will be passed the full response before it is passed through, and which returns an object that defines how the the response content should before it's returned to the client.

The callback should return an object that definies how the response should be changed. All fields on the object are optional. The possible fields are:

  • status (number, will replace the HTTP status code)
  • headers (object with string keys & values, replaces all headers if set)
  • body (string or buffer, replaces the body if set)
  • json (object, to be sent as a JSON-encoded body, taking precedence over body if both are set)
